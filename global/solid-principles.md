# Principios SOLID
1. **Single Responsibility**: Una clase/función, un solo motivo para cambiar.
2. **Open/Closed**: Abierto a extensión, cerrado a modificación.
3. **Liskov Substitution**: Los subtipos deben ser sustituibles por sus tipos base.
4. **Interface Segregation**: Interfaces pequeñas y específicas, no genéricas.
5. **Dependency Inversion**: Depender de abstracciones, no de concreciones.
